MIT6.828 Operating System hw system call

Introduction

UCI CS143A and CS238P, Operating System, are courses refered to MIT6.828.
This homework is to implement a system call, dump, which can dump memory of a process to screen, and use user program to test.
Detailed hw requirement:
hw3 system call

basic settings

  • to add a system call we need to modify a series of files in xv6, following:
    add sytem call to xv6
  • add a user program and modify Makefile:
    add user program to xv6

main part of system call is implemented in proc.c

proc.c

Arguments

  • pid: pid of process to be dumped
  • addr: virtual address of process
  • buffer
  • size: copy from memory to buffer based on page

Implementation

  • for loop ptable, use pid to find related pgdir(see struct proc in proc.h) -> get pgdir
  • call walkpgdir() and pass pgdir to it -> get pte
  • pa = PTE_ADDR(*pte) -> get physical address
  • turn pa to va, call memmove() and pass va to it -> move memory to buffer

walkpgdir(pgdir, va, a)

  • pgdir: pointer to page directory
  • va: virtual address will be round to the beginning of one page

test program

dump.c

Implementation

  • for loop: call dump(), pass in addr(increase PGSIZE each time), until dump() return -1
  • print content in buffer
